Detailed Description

DTLS processor that performans the DTLS handshake and encrypts/decrypts buffers once the handshake has succeeded. The object exposes a "user side" data source and data sink to communicate "user data" (typically an application or a library like CoAP), and a "transport side" data source and data sink to connect to a "transport" (typically a GG_DatagramSocket) that will transmit and receive the DTLS records. The object uses a GG_TimerScheduler to create retransmission timers when needed.
